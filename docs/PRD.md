# 사회인 야구 AI 플랫폼 — "더그아웃(Dugout)"

**문서 버전**: v1.0  
**작성일**: 2026.03.19  
**상태**: Draft

## 1. 개요

### 1-1. 배경

한국 사회인 야구 시장은 등록팀 약 3만 팀, 등록 선수 50만 명 이상 규모이다. 현재 사회인 야구팀의 운영은 대부분 **네이버 밴드**와 **카카오톡 단체 채팅방**에 의존하고 있으며, 기록 관리는 게임원(GAMEONE)이 사실상 독점하고 있다.

**현재 팀 운영 방식의 문제점:**

- **네이버 밴드**: 투표 기능으로 출석 확인, 일정 공유, 사진 공유 등에 활용되지만 야구에 최적화되어 있지 않다. 투표 결과를 보고 주장이 수동으로 라인업을 짜야 하며, 회비 관리는 별도 엑셀이나 메모로 처리한다.
- **카카오톡**: 빠른 소통에는 유리하지만, 중요 공지가 대화에 묻히고, 출석 확인이 체계적이지 않다. "저 이번 주 참석이요", "저 불참이요" 같은 메시지를 주장이 일일이 취합해야 한다.
- **게임원**: 기록 플랫폼이지 팀 운영 도구가 아니다. 과금 논란(2022~2023)으로 대안을 찾는 움직임이 활발하다.

어떤 서비스도 "팀 운영 자동화"와 "AI 기반 매칭"을 제공하지 않고 있으며, 이것이 본 제품의 핵심 기회이다.

### 1-2. 제품 비전

> **밴드와 카카오톡에 흩어진 팀 운영을 하나로 모으고, AI로 자동화하는 사회인 야구 전용 플랫폼**

### 1-3. 미션 스테이트먼트

사회인 야구팀의 주장/매니저가 매주 반복하는 번거로운 팀 운영 업무(출석 확인, 라인업 작성, 회비 관리)를 AI가 대신하고, 팀 간·용병 매칭을 지능적으로 연결하여 사회인 야구 생태계를 혁신한다.

## 3. 타겟 사용자

### 3-1. 페르소나

**페르소나 A: 김주장 (32세, 팀 주장)**
- 3년차 사회인 야구 팀 주장, 팀원 25명
- 매주 금요일 밤 밴드에 투표 올리고, 토요일 아침까지 미응답자에게 카톡 보냄
- 토요일 새벽에 출석 인원 보고 라인업을 급하게 짬
- 회비 미납자 독촉이 가장 스트레스

**페르소나 B: 박선수 (28세, 일반 선수)**
- 2개 팀에 소속, 주말마다 경기
- 밴드 알림을 꺼놔서 투표를 자주 놓침
- 라인업을 경기 당일에야 확인해서 포지션 준비가 안 됨

**페르소나 C: 이용병 (35세, 무소속 용병)**
- 고정 팀 없이 주말마다 용병으로 뛰고 싶음
- 밴드 게시판, 게임원 용병 게시판을 매일 확인하지만 자기 지역·포지션에 맞는 건 드묾
- 가끔 실력 차이가 너무 나는 경기에 배정되어 불만

### 3-2. 사용자 여정 (As-Is vs To-Be)

**경기 준비 과정 — 주장 관점:**

```
[As-Is: 밴드 + 카카오톡]

월요일    밴드에 "이번 주 토요일 경기 투표" 글 등록
화~목     밴드 미응답자에게 카톡 개별 연락 (5~10명)
금요일    밴드 투표 결과 확인 → 참가자 목록 수기 정리
금요일밤  참가자 보고 라인업을 메모장에 작성
토요일새벽 카톡 단톡방에 라인업 사진 공유
토요일아침 "저 늦어요" "저 결국 못가요" 카톡 → 라인업 급수정
경기 후   회비 개별 입금 요청 카톡, 미납자 추적

[To-Be: 더그아웃]

월요일    앱에서 경기 일정 등록 (2탭)
          → 자동 푸시 알림 발송 + 밴드/카톡 알림 연동
          → AI가 과거 패턴으로 "예상 참가 16명" 표시
화~목     미응답자에게 자동 리마인드 (48h, 24h 전)
금요일    출석 확정 → AI가 라인업 자동 추천
          → 주장이 드래그&드롭으로 미세 조정
금요일밤  라인업 확정 버튼 → 전 멤버에게 자동 알림
토요일    변동 시 실시간 라인업 수정 & 알림
경기 후   회비 자동 집계, 미납 알림 자동 발송
```

#### F1. 인증 & 온보딩

| 항목 | 내용 |

| ID | F1 |
| 우선순위 | P0 (필수) |

- F1-3: Apple 로그인 (iOS 필수)
- F1-4: Google 로그인 — 보조 채널 (Gmail/안드로이드 사용자 대응)

- Apple: 클라이언트가 Identity Token(JWT)를 받아 서버에 전달 → 서버가 Apple JWKS로 서명 검증 후 sub/email 추출

#### F2. 팀 생성 & 멤버 관리

| 항목 | 내용 |
|------|------|
| ID | F2 |
| 우선순위 | P0 (필수) |
| 설명 | 팀 생성, 멤버 초대, 역할 관리 |

**상세 요구사항:**
- F2-1: 팀 생성 (팀명, 로고, 지역, 부수(1~4부), 홈구장, 활동요일/시간)
- F2-2: 초대 방식
  - 초대 링크 생성 → 카카오톡/밴드로 공유 가능
  - QR 코드 초대 (오프라인 모임에서 사용)
  - 팀 코드 입력 방식
- F2-3: 멤버 프로필 (주 포지션, 서브 포지션(복수), 등번호, 타석, 투타)
- F2-4: 역할 체계
  - 주장: 모든 권한
  - 매니저: 일정/라인업/회비 관리
  - 회계: 회비 관리
  - 일반: 출석 투표, 조회
- F2-5: 밴드 그룹과의 병행 안내
  - "기존 밴드는 유지하면서, 출석/라인업/회비만 더그아웃으로" 가이드

**밴드/카톡 전환 전략:**
> 사용자에게 밴드를 당장 버리라고 하지 않는다. 밴드의 사진 공유·자유 소통은 유지하되, 출석 투표·라인업·회비 관리의 "공식 창구"만 더그아웃으로 옮기도록 유도한다. 전환의 핵심 동력은 앱 푸시 알림(FCM)과 자동화된 운영 가치(AI 추천·자동 리마인드)이며, 카카오 알림톡은 비용 부담으로 영구 제외한다.

#### F3. 경기 일정 & 출석 관리

| 항목 | 내용 |
|------|------|
| ID | F3 |
| 우선순위 | P0 (필수) |
| 설명 | 밴드 투표를 대체하는 출석 관리 시스템 |

**상세 요구사항:**
- F3-1: 경기 일정 등록
  - 날짜, 시간, 구장 (DB 선택 or 직접 입력), 상대팀 (선택)
  - 반복 일정 설정 (매주 토요일 8시 등)
  - 집합 시간 / 경기 시간 분리
- F3-2: 출석 투표
  - 상태: 참가 ✅ / 불참 ❌ / 미정 ❓ / 늦참 ⏰ / 조퇴 🚪 (밴드에 없는 기능)
    - **조퇴**: 경기 시작은 참석하나 중간에 먼저 이탈해야 하는 경우 (라인업 편성 시 교체 예정자로 반영)
  - 사유 입력 (선택)
  - 투표 마감 시간 설정 가능
  - 실시간 참가 현황 대시보드 (참가 N명, 불참 N명, 미응답 N명)
  - 부분 참가자 구분 표시 (늦참/조퇴 N명)
- F3-3: 자동 리마인드 알림
  - 경기 48시간 전: 미응답자에게 1차 알림
  - 경기 24시간 전: 미응답자에게 2차 알림 (어조 강화)
  - 알림 채널: 앱 푸시 (FCM)
  - 주장에게 현황 요약 알림 ("현재 12명 참가, 5명 미응답")
- F3-4: **🤖 AI 출석 예측**
  - 미응답자의 예상 참가 확률 표시
  - 전체 예상 참가 인원 표시 ("예상 참가: 14~16명")
  - 근거 표시 ("김OO: 토요일 출석률 90%, 참가 예상")
  - 경기 취소 위험 알림 ("예상 인원 9명 미만, 용병 모집을 추천합니다")

**밴드 투표 대비 차별점:**
- 밴드: 참가/불참 2가지 → 더그아웃: 참가/불참/미정/늦참/조퇴 5가지
- 밴드: 수동 리마인드 → 더그아웃: 자동 리마인드 + AI 예측
- 밴드: 결과를 주장이 수동 취합 → 더그아웃: 실시간 대시보드
- 밴드: 투표만 → 더그아웃: 투표 → 라인업 → 알림까지 자동 연결

#### F5. 회비 관리

| 항목 | 내용 |
|------|------|
| ID | F5 |
| 우선순위 | P1 (중요) |
| 설명 | 팀 회비 및 경기별 비용 관리 |

**상세 요구사항:**
- F5-1: 회비 설정
  - 정기 회비 (월/분기/시즌 단위)
  - 경기별 추가 비용 (구장비, 심판비, 공값 등)
  - 멤버별 차등 금액 설정 가능
- F5-2: 납부 현황 대시보드
  - 멤버별 납부/미납 상태
  - 전체 수입/지출 요약
  - 잔액 표시
- F5-3: 미납 알림
  - 납부 기한 전후 자동 알림 (앱 푸시)
  - 주장에게 미납자 리스트 요약 알림
- F5-4: 경기별 정산
  - 경기 후 자동 비용 산정 (참가 인원으로 나누기)
  - 용병 참가비 별도 관리

#### F6. 연습경기 매칭

| 항목 | 내용 |
|------|------|
| ID | F6 |
| 우선순위 | P1 (중요) |
| 설명 | AI 기반 팀 간 연습경기 매칭 |

**상세 요구사항:**
- F6-1: 매칭 요청 등록
  - 희망 날짜/시간대
  - 선호 구장 or 지역
  - 희망 상대 수준 (1~4부 or 레이팅 범위)
  - 홈/원정/무관 선택
- F6-2: **🤖 AI 매칭 추천**
  - 매칭 스코어 기반 추천 리스트 (최대 5팀)
  - 각 추천에 매칭 적합도 표시 ("실력 유사도 92%, 거리 3.2km")
  - 매칭 요소: 실력 레이팅, 지리적 거리, 시간대 호환, 매너 점수
- F6-3: 매칭 플로우
  - 요청 → AI 추천 → 상대팀 선택 → 상대팀 수락/거절 → 매칭 성사
  - 양 팀 주장 간 인앱 채팅 (구장·시간 세부 조율)
  - 매칭 성사 시 양 팀 멤버에게 알림
- F6-4: 경기 결과 & 평가
  - 스코어 입력 (양 팀 모두 입력 → 교차 검증)
  - 상대팀 매너 평가 (1~5점 + 태그: 시간 잘 지킴, 페어플레이 등)
  - 결과 반영 → 팀 레이팅 자동 업데이트

#### F9. 알림 시스템

| 항목 | 내용 |

**상세 요구사항:**
- F9-1: 알림 채널
  - 앱 푸시 알림 (FCM, Phase 3-C 도입)
  - 이메일 (선택, 후속 Phase)
- F9-2: 알림 유형별 발송 규칙

| 알림 유형 | 발송 시점 | 푸시 | Phase |
|-----------|-----------|------|-------|
| 라인업 확정 | 주장 확정 시 | ✅ | 3-C (완료) |
| 새 경기 일정 | 일정 등록 즉시 | ✅ | 3-D (완료) |
| 출석 리마인드 | 경기 48h, 24h 전 (정시 cron, KST) | ✅ | 3-D (완료) |
| 출석 응답 변경 (주장) | 가용성 경계가 바뀐 경우만 | ✅ | 3-D (완료) |
| 라인업 변경 | 변경 시 | ✅ | 후속 Phase |
| 회비 청구 | 청구일 | ✅ | 후속 Phase |
| 회비 미납 리마인드 | 기한 후 3일, 7일 | ✅ | 후속 Phase |
| 매칭 요청 수신 | 요청 시 | ✅ | 후속 Phase |
| 매칭 성사 | 성사 시 | ✅ | 후속 Phase |
| 용병 추천 | 조건 맞는 경기 등록 시 | ✅ | 후속 Phase |

- F9-3: 알림 설정 (Phase 3-D 완료)
  - 유형별 on/off (4종: 매치 등록 / 라인업 확정 / 출석 리마인드 / 출석 변경)
  - 방해 금지 시간 (DnD, 기본 22:00–08:00, 자정 넘김 지원)
  - DnD 는 **출석 리마인드에만 적용** — 즉시성 알림 3종(매치 등록 / 라인업 확정 / 출석 변경)은 DnD 무시. 리마인드는 DnD 종료 후 첫 정시 cron 에 재시도.
  - MyPage → "알림 설정" 진입.

- F9-4: 푸시 탭 → 화면 이동 (Phase 3-D 완료)
  - 알림 탭 시 일정 탭 활성 + 해당 경기 상세 화면 자동 진입.
  - 앱 종료 상태에서 탭한 경우(cold-start)도 동일 동작 (`DeepLinkInbox` 가 pending 보관 후 첫 진입 시 소비).

**FCM 우선 전략:**
> Phase 3-C 에서 FCM 인프라 도입, Phase 3-D 에서 알림 종류 확대 + 사용자 설정 + deeplink 까지 완성. 카카오 알림톡은 발송당 과금 비용 + 비즈니스 채널 심사 부담으로 영구 제외. 결과적으로 앱 미설치자 도달은 포기하지만, 앱 설치 사용자의 알림 경험에 집중하여 무료/즉시/Android 호환의 이점을 확보. 사용자 전환은 앱 설치 자체를 유도하는 마케팅/온보딩 흐름으로 대체.

## 6. 밴드/카카오톡 마이그레이션 전략

사용자가 기존 소통 채널을 갑자기 바꾸기 어렵다는 점을 인정하고, **점진적 전환**을 유도한다.

### Stage 1: 보조 도구 (출시 초기)
- "밴드는 그대로 쓰시고, 출석 투표만 더그아웃에서 해주세요"
- 앱 푸시 알림(FCM)으로 투표 리마인드 → 밴드 투표를 자연스럽게 대체
- 라인업 카드 이미지를 카카오톡 등 외부 채널로 공유할 수 있게 지원 (`UIActivityViewController`)

### Stage 2: 핵심 도구 (3개월 후)
- 회비 관리, 매칭 기능까지 확장
- 팀 공지를 더그아웃에서 작성 → 앱 푸시로 전체 멤버 알림
- "밴드에서 사진 공유, 더그아웃에서 팀 운영" 이원 체제

### Stage 3: 주 플랫폼 (6개월 후)
- 인앱 소통 기능 강화
- 경기 기록 기능 추가
- 밴드를 완전히 대체할 수 있는 수준
