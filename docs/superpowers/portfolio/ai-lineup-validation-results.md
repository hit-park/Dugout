# AI 라인업 추천 — 품질 검증 결과 (포트폴리오)

> 정식 제품 문서(PRD/TDD)와 분리된 포트폴리오 산출물.

## 무엇을 검증했나
세이버매트릭스 타순 엔진(The Book lite + shrinkage k=50)을 **특성을 심은 합성 데이터**로 검증.

- The Book 슬로팅: 종합 최고타자 → 2번, 순수 출루형 → 1번, 거포 → 4번 (큰 표본에서 정확 슬롯 단언).
- shrinkage: 작은 표본(5타석)일수록 보정 지표가 팀 평균으로 수축(극단값 불신), 큰 표본(200타석)에서 또렷.
- 콜드 스타트: 기록 0이면 좌우 교차 폴백(`order`가 `None`).
- 반증 가능성: 엔진의 슬로팅 순서를 일부러 바꾸면 검증 테스트가 실패함을 확인.

## 재현
```
cd dugout-ai && pytest tests/test_lineup_validation.py -v
python -m scripts.demo_lineup            # 합성 PA=200 vs PA=5
python -m scripts.demo_lineup --club-idx 40837   # 실데이터(마스킹)
```

> 데모는 `python scripts/demo_lineup.py`가 아니라 `python -m scripts.demo_lineup`로 실행한다.
> dugout-ai는 설치형 패키지가 아니라, 스크립트를 직접 실행하면 프로젝트 루트가 `sys.path`에
> 올라가지 않아 `app` import가 실패한다. `-m`는 현재 디렉토리를 경로에 올려 이를 해소한다.

## 합성 데모 출력 (PA=200 vs PA=5)
큰 표본에서는 아키타입 특성이 또렷하고(출루 .490 / 순장타 .305), 작은 표본에서는
shrinkage가 같은 선수의 지표를 팀 평균 쪽으로 끌어당겨 라인업이 평탄해진다.

```
## 합성 PA=200 (특성 또렷)
1번  PURE_ONBASE#2    — 출루율 .422로 1번(테이블세터) 배치
2번  OVERALL#1        — 종합 최고타자로 2번 배치 (출루 .490/순장타 .332)
3번  AVERAGE#4        — 종합 상위 타자로 3번 배치
4번  POWER#3          — 장타력(ISO .305)으로 4번 배치
...

## 합성 PA=5 (작은 표본 — shrinkage로 평탄화)
2번  OVERALL#1        — 종합 최고타자로 2번 배치 (출루 .279/순장타 .116)
4번  POWER#3          — 장타력(ISO .148)으로 4번 배치
...
```

PA=200의 OVERALL 출루 .490 / POWER ISO .305가 PA=5에서 .279 / .148로 수축한다 —
표본이 작을수록 엔진이 극단값을 덜 믿는다는 shrinkage의 직접 증거.

## 한계 (정직하게)
- 합성 데이터는 "엔진이 설계 규칙대로 동작하는가"를 검증한다. "사회인 야구에서 이 타순이
  득점을 늘리는가"는 실경기 결과가 쌓여야 검증 가능 — 별도 과제.
- gameone 표본은 경기수가 작아 대부분 shrinkage 폴백 구간. 실데이터는 "파이프라인이
  진짜로 돈다"의 증거이지, 타순 효과의 증거는 아니다.
- gameone 실데이터 스모크 테스트는 네트워크가 없으면 skip된다(품질이 아닌 파이프라인 검증).
